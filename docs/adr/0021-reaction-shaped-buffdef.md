# Reaction-shaped `BuffDef`: no-instance dispatch via absent `target` + `duration`

A `BuffDef` whose `target` and `duration` are both omitted is a **reaction**: it has no instance in the store, no lifecycle, and no `buffApplied`/`buffRefreshed`/`buffExpired`/`buffConsumed` log events. Its effects (constrained to `resource | emitHit | coordHit`) dispatch when the trigger matches and any `condition` passes. A `BuffDef` with both fields present is a **stateful buff**, behaving exactly as today. The schema stays a single `BuffDef` type — the two flavors are distinguished by structural absence, not a discriminator tag, and the both-or-neither invariant is enforced at data-load time. This closes the long-standing anti-pattern of authoring `duration: { kind: "frames", v: 1 }` as a fake one-frame buff just to hang a dispatch off a trigger — a pattern that flooded the simulation log with start/end pairs and lied about the model (no presence existed, no state persisted, the buff was a vehicle for the trigger). The pattern was widespread: forte/concerto/energy grants, Verina's coord reactions, Sanhua's three Detonate bursts (which encoded the same shape with `permanent` instead of `frames: 1`).

## Considered Options

- **Log-side filter: hide `buffApplied`/`buffExpired` rows when applied-frame == expired-frame.** Rejected. Suppresses the symptom while entrenching the anti-pattern. New authors keep writing `frames: 1` because the cost is now invisible, and the data layer continues to lie about what these definitions model. Bandaid.
- **Per-BuffDef `silent: true` opt-out on `BuffDef`.** Rejected. Same as above — preserves the lie, just adds an opt-out flag the author has to remember. Vocabulary cost without a model fix.
- **Sibling `ReactionDef` type, separate `reactions: [...]` array on the character/weapon file.** Rejected. Two arrays in the data file fork what is conceptually one list of "things this character reacts to or grants." A reader scanning the file has to mentally merge two lists; tooling that walks BuffDefs has to walk both. The reaction is a _flavor_ of buff, not a parallel construct.
- **`kind: "stateful" | "reaction"` discriminator tag on `BuffDef`.** Rejected. The engine doesn't benefit — cooldown is already keyed per-`(def.id, sourceCharacterId)` at candidate-selection, and the instance-or-not branch reads `duration == null` just as easily as `kind === "reaction"`. The tag adds a field every existing buff needs to backfill (or default), and pulls the model back toward "two shapes under one name." Absence-inference plus a load-time validator gives the same guarantee with less surface area.
- **TS-discriminated union under one name: `BuffDef = StatefulBuffDef | ReactionBuffDef`.** Rejected for the same reason as the discriminator tag — same split, just hidden in the type system. Author still reads two shapes. We trade compile-time enforcement for one extra load-time validator; the validator runs in CI on every data change, so the feedback loop is equivalent.
- **Lift `target` onto each `ResourceEffect`.** Rejected. The motivating case (a team-scoped resource grant via reaction) is already expressible by keeping `target` at the `BuffDef` level for reactions too — the field is meaningful for both flavors (scopes effects for reactions, scopes the buff itself for stateful). Per-effect target would create a redundant second target site on stateful buffs and conflict with the existing `damage.target` already on heal-emits.
- **Keep `frames: 1` as the dispatch idiom and rename it (e.g. `duration: { kind: "instant" }`).** Rejected. Preserving the buff-shaped scaffolding muddies what `target` and `duration` mean on real stateful buffs. The fields earn their keep when they describe a window the buff actually occupies; faking that window for dispatch dilutes their semantics.
- **Sanhua's three `*-burst` defs (`duration: permanent` + emit-only) stay as buffs; only the `frames: 1` set migrates.** Rejected. The `permanent` encoding was incidental — the bursts have no presence semantics, no stat effects, no queryable state. Same dispatch-only shape as the `frames: 1` set; consistency wins. Leaving them as `permanent` buffs would leave two encodings of the same concept in the data.
- **Eager condition evaluation (per ADR-0011) keyed on "all effects are immediate."** Was the existing predicate; rejected going forward. After migration the predicate `effects.every(e => kind ∈ {resource, emitHit, coordHit})` and `def.duration == null` describe the same set. The latter is explicit about _why_ the eager path applies (no instance exists for lazy re-evaluation), the former relies on a derived property. Switch to `duration == null`.

## Consequences

- **Schema**: `BuffDef.target` and `BuffDef.duration` become optional. Both present → stateful (current behavior). Both absent → reaction. One present without the other → load-time error.
- **Effect constraint on reactions**: a `BuffDef` with `duration == null` containing a `stat` effect is a load-time error. Stat effects need a duration window to be meaningful; without one the modifier has no scope.
- **Load-time validator** (`validateBuffDef` in the data loader): enforces `(target == null) === (duration == null)`; if `duration == null` then no `stat` effect; reactions cannot declare `stacking` or `consumedBy` (both are instance-bound).
- **Engine — instance creation**: at the buff-apply phase, skip instance creation when `def.duration == null`. The `resource`, `emit`, and `coordHit` phases already iterate the same `candidates` set and dispatch independently of whether the candidate becomes an instance, so reactions fall out by absence — no new dispatch primitive.
- **Engine — eager condition**: the predicate at the candidate-filter step (currently keyed on `effects.every(e => kind ∈ {resource, emitHit, coordHit})`) simplifies to `def.duration == null`. After migration these are equivalent; the new form is explicit.
- **Cooldown semantics unchanged**. `cooldownLastFired` is already a `Map<string, number>` keyed `${def.id}|${sourceCharacterId}`, enforced at candidate-selection before any instance branch. Works identically for both flavors.
- **Log behavior**: no `buffApplied`/`buffRefreshed`/`buffExpired`/`buffConsumed` events emit for reactions (no instance, no lifecycle to report on). Synthetic events the reaction emits (`HitEvent`, `SustainEvent`, `ResourceChange`) log exactly as before.
- **Migration sweep**: 13 BuffDefs migrate to reaction-shape — drop `target` and `duration` (and `stacking` where present):
  - `weapons/variation.ts` — `Ceaseless Aria` (resource concerto, retains `cooldown: 20`).
  - `characters/encore.ts` — `s2.sheep-counting-lullaby` (resource energy, retains `cooldown: 10`; drops `stacking: { max: 1, onRetrigger: "refresh" }` as redundant).
  - `characters/sanhua.ts` — `sequence.s4.energy`; `ice-thorn-burst`, `ice-prism-burst`, `ice-glacier-burst` (the last three drop `duration: permanent` along with `target`).
  - `characters/verina.ts` — `lib.mark-coord-reaction`, `s6.joyous-harvest-coord`, `forte.grant-b5`, `forte.grant-skill`, `forte.grant-intro`, `forte.starflower-consume`, `s2.concerto-grant`.
- **Stays stateful** (`stat` effect needs a window): `verina.s6.joyous-harvest-dmg`. Sanhua's S5 Crit DMG is one `sanhua.s5.unraveling-fate` buff — a `requiresSequence:5` permanent buff that folds +100% Crit DMG into the three Ice Burst emits via `appliesToHits` (ADR-0029); it carries no time window because the bonus is hit-scoped, not frame-scoped.
- **Stays stateful** (presence-only, queried via `buffActive`): `sanhua.flag.ice-thorn`/`.ice-prism`/`.ice-glacier` (also have `consumedBy`); `verina.lib.photosynthesis-mark`. The flag pattern is unchanged.
- **ADR-0020 amendment**: the two-buff coord pattern's reaction half no longer carries `target: self` + `duration: { kind: "frames", v: 1 }`. The presence-flag BuffDef is unchanged. ADR-0020's data-migration note for `char.verina.lib.mark-coord-reaction` is now: drop `target`/`duration`, keep the two `coordHit` effects. The ADR-0020 prescription that says coord-reactions are authored as a `BuffDef` with `frames: 1` becomes "...authored as a reaction-shaped `BuffDef`."
- **ADR-0011 amendment**: the closing paragraph on which defs get eager-evaluated conditions reframes from "all-immediate effects" to `duration == null`. Behavior unchanged after migration.
- **New domain term**: **Reaction** (a reaction-shaped `BuffDef`). Synonyms considered and rejected: "reactor" (collides with the in-game term Resonator), "listener" / "handler" / "hook" (read as infrastructure, not game-modeling). One term across the codebase: reaction.
